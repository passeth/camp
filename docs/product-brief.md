# Camp Product Brief

Source conversation: https://chatgpt.com/share/6a315358-29f4-83ee-acaa-ba44fa945e00

## Concept

Camp is an AI-powered study magazine, wiki, and archive platform for study members. Public visitors can read and share published content. Logged-in members can write, manage drafts, and use AI-assisted workflows. Admins can approve members, publishing, categories, and future sync/deploy jobs.

## Core content areas

- Member profiles and member-specific press pages
- Press / magazine posts
- Daily Review: Hermes summarizes daily messenger conversations
- Topic learning notes with folder/subfolder navigation
- Study Logs for meeting notes and outcomes
- Teach Pages generated by Hermes as HTML or interactive learning content

## Architecture direction

Start with a normal Vercel web app and add Hermes/VPS integration later.

```text
Vercel
- Frontend/app hosting
- Public and member-only web UI

GitHub
- Website source code
- Markdown/MDX content source

Supabase
- Auth
- Member roles
- Comments, reactions, views
- Publish workflow metadata
- Optional Storage for images

VPS later
- Hermes Agent
- Obsidian Vault workspace
- Sync/publish jobs
```

## Storage model

- Markdown/body content: Git-backed content files first, later Obsidian Vault sync.
- Supabase: auth, profiles, roles, comments, reactions, views, metadata, publish state.
- Images/assets: Supabase Storage, S3, or Cloudflare R2. Avoid storing large images in Git.

## Publish workflow guardrail

Browsers must not directly use GitHub tokens or run Git pushes.

Recommended future flow:

```text
Member clicks publish
→ server API checks auth/role
→ API updates publish state or creates job
→ VPS sync/publish job updates Markdown/frontmatter
→ VPS commits and pushes to GitHub
→ Vercel deploys
→ UI shows status
```
